[FastMCP Cloud](https://fastmcp.cloud) 是由 FastMCP 团队构建的托管 MCP 服务器平台。虽然 FastMCP 框架将始终保持完全开源，但我们创建了 FastMCP Cloud 来解决开发人员面临的部署挑战。我们的目标是提供最快的方式让您的 MCP 服务器能够被 Claude 和 Cursor 等 LLM 客户端使用。 

FastMCP Cloud 是一个年轻的产品，我们欢迎您的反馈。请加入我们的 [Discord](https://discord.com/invite/aGsSC3yDF4) 分享您的想法和建议，您可以期待每周都能看到新功能和改进。


<Note>
FastMCP Cloud 支持 **FastMCP 2.0** 服务器，也支持使用官方 MCP Python SDK 创建的 **FastMCP 1.0** 服务器。
</Note>

<Tip>
FastMCP Cloud 在 beta 期间完全免费！
</Tip>

## 先决条件

要使用 FastMCP Cloud，您需要一个 [GitHub](https://github.com) 账户。此外，您需要一个包含 FastMCP 服务器实例的 GitHub 仓库。如果您还不想创建一个，可以继续进行[步骤 1](#step-1-create-a-project)并使用 FastMCP Cloud 快速入门仓库。 

您的仓库可以是公开的或私有的，但必须至少包含一个包含 FastMCP 服务器实例的 Python 文件。 
<Tip>
要确保您的文件与 FastMCP Cloud 兼容，您可以运行 `fastmcp inspect <file.py:server_object>` 来查看 FastMCP Cloud 运行您的服务器时会看到什么。
</Tip>

如果您的仓库中有 `requirements.txt` 或 `pyproject.toml`，FastMCP Cloud 会自动检测您服务器的依赖项并为您安装它们。请注意，您的文件*可以*有 `if __name__ == "__main__"` 块，但它会被 FastMCP Cloud 忽略。

例如，一个最小的服务器文件可能如下所示：

```python
from fastmcp import FastMCP

mcp = FastMCP("MyServer")

@mcp.tool
def hello(name: str) -> str:
    return f"Hello, {name}!"
```

## 入门指南

向 FastMCP Cloud 部署服务器只需要三个步骤：

### 步骤 1：创建项目

访问 [fastmcp.cloud](https://fastmcp.cloud) 并使用您的 GitHub 账户登录。然后，创建一个项目。每个项目对应一个 GitHub 仓库，您可以从自己的仓库创建一个项目，或使用 FastMCP Cloud 快速入门仓库。

<img src="/assets/images/fastmcp_cloud/quickstart.png" alt="FastMCP Cloud Quickstart Screen" />

接下来，系统将提示您配置项目。

<img src="/assets/images/fastmcp_cloud/create_project.png" alt="FastMCP Cloud Configuration Screen" />

配置界面允许您指定：
- **名称**：您项目的名称。这将用于为您的服务器生成唯一的 URL。
- **入口点**：包含您的 FastMCP 服务器的 Python 文件（例如，`echo.py`）。此字段与 `fastmcp run` 命令具有相同的语法，例如 `echo.py:my_server` 来指定文件中的特定对象。
- **身份验证**：如果禁用，您的服务器对公众开放。如果启用，只有您 FastMCP Cloud 组织的其他成员才能连接。

请注意，FastMCP Cloud 会自动从 `requirements.txt` 或 `pyproject.toml` 文件中检测您服务器的 Python 依赖项。

### 步骤 2：部署您的服务器

一旦您配置了项目，FastMCP Cloud 将：
1. 克隆仓库
2. 构建您的 FastMCP 服务器
3. 将其部署到唯一的 URL
4. 立即使其可用于连接

<img src="/assets/images/fastmcp_cloud/deployment.png" alt="FastMCP Cloud Deployment Screen" />

FastMCP Cloud 会监控您的仓库，并在您向 `main` 分支推送更改时重新部署您的服务器。此外，FastMCP Cloud 会为您打开的每个 PR 构建和部署服务器，将它们托管在唯一的 URL 上，这样您就可以在更新生产服务器之前测试更改。

### 步骤 3：连接到您的服务器

一旦您的服务器部署完成，它将可通过类似以下的 URL 访问：

```
https://your-project-name.fastmcp.app/mcp
```

一旦看到部署成功，您应该就能立即连接到它！FastMCP Cloud 为热门的 LLM 客户端提供即时连接选项：

<img src="/assets/images/fastmcp_cloud/connect.png" alt="FastMCP Cloud Connection Screen" />
