Model Context Protocol（MCP）是一个开放标准，旨在解决 AI 开发中的一个基础问题：大语言模型（LLM）如何可靠且安全地与外部工具、数据和服务交互？

它是**概率性、非确定性的 AI 世界与确定、可靠的代码和数据世界之间的桥梁。**

你当然可以为 LLM 构建自定义 REST API，但 MCP 为 AI 原生通信提供了专门且标准化的“端口”。可以把它理解为 **AI 的 USB-C**：一个单一且定义明确的接口，用于连接任何符合规范的 LLM 与任何符合规范的工具或数据源。

本指南将对协议本身做高层概览。我们会使用 **FastMCP** 这个领先的 MCP Python 框架，通过简单代码示例说明这些概念。

## 为什么需要协议？

既然已经有无数 API，最常见的问题自然是：“为什么还需要另一个？”

答案在于**标准化**。AI 生态是碎片化的。每个模型提供商都有自己定义和调用工具的方式。MCP 的目标是创建一种通用语言，并带来几个关键优势：

1.  **互操作性：** 构建一个 MCP 服务端，它就能被任何符合 MCP 的客户端（Claude、Gemini、OpenAI、自定义 agent 等）使用，无需自定义集成代码。这是该协议最重要的承诺。
2.  **可发现性：** 客户端可以在运行时动态询问服务端具备哪些能力。它们会收到一份结构化、机器可读的工具和资源“菜单”。
3.  **安全性：** MCP 提供了清晰的沙箱边界。LLM 不能在你的服务端执行任意代码；它只能*请求*运行你明确暴露的、带类型且经过验证的特定函数。
4.  **可组合性：** 你可以构建小型、专用的 MCP 服务端，并将它们组合起来创建强大而复杂的应用。

## MCP 核心组件 

MCP 服务端通过三个主要组件暴露能力：工具、资源和提示词。

### 工具：可执行操作

工具是 LLM 可以请求服务端执行的函数。它们是 MCP 中面向操作的部分。

借用 REST API 的类比，你可以把**工具看作 `POST` 请求。** 它们用于*执行操作*、*改变状态*或*触发副作用*，例如发送电子邮件、向数据库添加用户或进行计算。

使用 FastMCP，创建工具就像装饰一个 Python 函数一样简单。

```python
from fastmcp import FastMCP

mcp = FastMCP()

# 此函数现在是名为 "get_weather" 的 MCP 工具
@mcp.tool
def get_weather(city: str) -> dict:
    """获取特定城市的当前天气。"""
    # 在真实应用中，这里会调用天气 API
    return {"city": city, "temperature": "72F", "forecast": "Sunny"}
```

### 资源：只读数据

资源是 LLM 可以读取的数据源。它们用于将信息加载到 LLM 的上下文中，为其提供训练数据中没有的知识。

继续使用 REST API 类比，**资源就像 `GET` 请求。** 它们的目的是以幂等方式*检索信息*，理想情况下不会产生副作用。资源可以是静态文本文件，也可以是数据库中的动态数据片段。每个资源都由唯一 URI 标识。

```python
from fastmcp import FastMCP

mcp = FastMCP()

# 此函数在 URI "system://status" 上提供一个资源
@mcp.resource("system://status")
def get_system_status() -> dict:
    """返回服务的当前运行状态。"""
    return {"status": "all systems normal"}
```

#### 资源模板

你也可以为动态数据创建**资源模板**。客户端可以请求 `users://42/profile` 来获取特定用户的资料。

```python
from fastmcp import FastMCP

mcp = FastMCP()

# 此模板为任意给定用户 ID 提供用户数据
@mcp.resource("users://{user_id}/profile")
def get_user_profile(user_id: str) -> dict:
    """返回特定用户的资料。"""
    # 从数据库获取用户...
    return {"id": user_id, "name": "Zaphod Beeblebrox"}
```

### 提示词：可复用指令

提示词是可复用、参数化的消息模板。它们提供了一种定义一致、结构化指令的方式，客户端可以请求这些指令来引导 LLM 完成特定任务。

```python
from fastmcp import FastMCP

mcp = FastMCP()

@mcp.prompt
def summarize_text(text_to_summarize: str) -> str:
    """创建一个提示词，要求 LLM 总结一段文本。"""
    return f"""
        Please provide a concise, one-paragraph summary of the following text:
        
        {text_to_summarize}
        """
```

## 高级能力

除了核心组件，MCP 还支持更高级的交互模式，例如服务端请求*客户端的* LLM 生成补全（称为**采样**），或服务端向客户端发送异步**通知**。这些特性支持更复杂的双向工作流，并且 FastMCP 完全支持它们。
